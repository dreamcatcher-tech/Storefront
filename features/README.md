# Storefront Feature Specs

These `.feature` files are agent-readable product/business specs, not Cucumber step-definition tests yet.

Allowed tags are intentionally small:

```text
@directory @card @agent @conscience @economics @api @governance @burn
```

Each feature should describe externally observable behavior: what a principal, agent, operator, provider, contributor, auditor, or Conscience should be able to see or verify.
